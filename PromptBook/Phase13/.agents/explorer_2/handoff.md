## 2. Logic Chain

1. **From Inputs to Component Architecture**:
   - The script JSON generated by Phase 12 contains narration text and visual parameters.
   - The Voice Production Engine must synthesize narration audio (Kokoro local/NPU, ElevenLabs cloud, OpenAI TTS cloud), normalize audio via EBU R128 (-14 LUFS), and emit a word-level timing manifest.
   - The Subtitle Generation Engine consumes the timing manifest and WAV audio to produce SRT, VTT, and ASS formatted subtitle tracks.
   - The Animation Production Engine uses timing durations from the voice engine and visual directives from the script JSON to render frame-accurate scene MP4 clips (Manim, Blender, Remotion).
   - The Video Assembly Engine uses FFmpeg to composite multi-track audio (voice, BGM, SFX), video scenes, and ASS subtitles into master 4K and upload 1080p MP4 files.
   - The Thumbnail Generation Service creates A/B image variants via Playwright SVG and Pillow engines.
   - The Publishing Manager handles YouTube Data API v3 upload, OAuth token refresh, chapter metadata, and scheduled release.
   - The Artifact Tracking & State Store enforces immutability via SHA-256 hashing and indexes all metadata in a relational database.

2. **From Component Architecture to Provider Abstraction**:
   - Hardcoding engines (e.g. only Manim or only Kokoro) creates vendor and platform lock-in.
   - Using Python `typing.Protocol` with `@runtime_checkable` allows structural subtyping without rigid inheritance.
   - The Provider Factory reads runtime YAML configuration (`media_production.yaml`), loads specified providers from `ProviderRegistry`, and injects settings.
   - The `FallbackProviderProxy` catches runtime exceptions from primary providers (e.g. ElevenLabs quota limit) and seamlessly fails over to secondary providers (e.g. Kokoro TTS).
